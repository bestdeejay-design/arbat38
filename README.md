# Arbat 38 - Система электронных ваучеров и ресторанный маркетплейс

## 📋 О проекте

Полнофункциональная система для автоматизации выдачи питания пассажирам при задержках рейсов и создания маркетплейса ресторанов в аэропортах.

**Заказчик:** ГК Арбат38  
**Исполнитель:** Аксиома  
**Версия:** 1.0

---

## 🎯 Два основных потока

### Поток 1: Система электронных ваучеров (MVP)
- Автоматизация выдачи питания при задержках рейсов
- Замена бумажных талонов на электронные QR-ваучеры
- 4 роли пользователей

### Поток 2: Ресторанный маркетплейс
- Предзаказ еды в ресторанах аэропортов
- Оплата картой + применение ваучеров
- Бонусная система
- Интеграция с iiko/R-Keeper

---

## 👥 Роли пользователей

1. **Пассажир** - мобильное приложение/PWA
2. **Сотрудник аэропорта** - веб-кабинет (создание ваучеров)
3. **Кассир точки питания** - веб-страница (сканирование QR)
4. **Менеджер ресторана** - веб-админка (управление меню и заказами)
5. **Администратор системы** - суперадминка (полный контроль)

## 📱 Интерактивные прототипы

### 1. Мобильное приложение пассажира
**Файл:** `mobile-app.html`

**Функционал:**
- Получение ваучера при задержке рейса
- Просмотр активного ваучера с QR-кодом
- Каталог ресторанов в аэропорту
- Меню ресторанов с фото и ценами
- Корзина с применением ваучера
- Оформление заказа
- Личный кабинет с бонусами

**Как открыть:**
```
Откройте файл mobile-app.html в браузере
```

### 2. Веб-кабинет сотрудника аэропорта
**Файл:** `airport-staff.html`

**Функционал:**
- Дашборд со статистикой
- Создание ваучеров на рейс
- Генерация QR-кода для стойки информации
- Список активных рейсов с задержкой
- Управление ваучерами
- Статистика по аэропортам

**Как открыть:**
```
Откройте файл airport-staff.html в браузере
```

### 3. Страница кассира
**Файл:** `cashier.html`

**Функционал:**
- Сканирование QR-кода ваучера (симуляция камеры)
- Ручной ввод кода ваучера
- Проверка статуса ваучера (действителен/использован/просрочен)
- Погашение ваучера
- Статистика погашений

**Как открыть:**
```
Откройте файл cashier.html в браузере
```

### 4. Админка менеджера ресторана
**Файл:** `restaurant-admin.html`

**Функционал:**
- Управление заказами (новый → принят → готовится → готов → выдан)
- Звуковые уведомления о новых заказах
- Управление меню (добавление/редактирование блюд)
- Управление категориями
- Настройки ресторана
- Статистика заказов

**Как открыть:**
```
Откройте файл restaurant-admin.html в браузере
```

## 🚀 Технологический стек (для продакшена)

### Backend
- **Node.js** + Express
- **PostgreSQL** - основная база данных
- **JWT** - аутентификация сотрудников
- **Redis** - кэширование и сессии

### Frontend
- **Mobile:** React Native или Flutter
- **Web:** React + TypeScript
- **PWA:** Для ваучеров (чтобы не устанавливать приложение)

### Интеграции
- **Платежи:** ЮKassa или Т-Банк
- **Push:** Firebase (Android), APNS (iOS)
- **POS:** iiko, R-Keeper API
- **QR:** Генерация на сервере

## 📊 Основные бизнес-правила

### Ваучеры
- 1 ваучер на 1 рейс на 1 тип (вода/еда)
- Действует 24 часа или до вылета
- Привязан к аэропорту
- Нельзя использовать на алкоголь

### Заказы
- Оплата картой или частично ваучером + карта
- Бонусы начисляются только на оплату картой
- Бонусы: 7% начисление, 50% списание
